# Glossary

| Term                | Meaning                                                                                                                                                                                                                                                                                         |
| ------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Raw Source**      | An unprocessed source document (article, paper, transcript, note). Immutable — the LLM only reads on ingest, never overwrites. Lives in `raw-sources/`. After source creation, no longer used by the LLM.                                                                                       |
| **Source**          | Wiki page of type `source` in the `sources/` folder. Processed, human-reviewed knowledge base from a Raw Source. All further processing (extraction, claims, updates) is based exclusively on Sources. Links to the Raw Source for human verifiability.                                       |
| **Wiki Page**       | An LLM-generated and maintained Markdown file in the vault with a fixed type (`source`, `entity`, `concept`, `synthesis`, `report`).                                                                                                                                                            |
| **Entity**          | Wiki page of type `entity`. Describes an identifiable thing — person, project, tool, organization, location, event. Collects claims.                                                                                                                                                            |
| **Concept**         | Wiki page of type `concept`. Describes an abstract idea, theory, pattern, method, or framework. Collects claims.                                                                                                                                                                                 |
| **Synthesis**       | Wiki page of type `synthesis`. Cross-sectional analysis linking multiple entities or concepts. Created from query or compile. Collects claims.                                                                                                                                                   |
| **Report**          | Wiki page of type `report`. Auto-generated dashboard. Completely regenerated on every compile.                                                                                                                                                                                                   |
| **Claim**           | Structured, verifiable assertion with unique `id:claim-xxx`, `conf`, `status`, and `*Evidence:*` (wikilink to a Source). The ID enables direct referencing from dashboards, index, and cross-references (format: `page-id#claim-id`). Turns prose into a trackable belief system.                |
| **Evidence**        | Proof for a claim, as a wikilink in the `*Evidence:*` field. Every claim carries its own provenance. No claim without evidence.                                                                                                                                                                 |
| **Human Block**     | Markdown section between `<!-- exolith:human:start -->` and `<!-- exolith:human:end -->`. The only explicitly marked area. Handwritten notes, never touched by the LLM. Everything outside is implicitly LLM-managed.                                                                          |
| **YAML Frontmatter** | Metadata block at the beginning of each page. Contains `id`, `page`, `title`, `status`, `tags`, `confidence`, `created`, `updated`.                                                                                                                                                             |
| **Ingest**          | The process of integrating a Raw Source into the wiki: read → discuss → source page → extraction (from the source) → update affected pages → trigger compile.                                                                                                                                   |
| **Compile**         | Reads the entire vault and regenerates index.md, backlinks, dashboards, and machine-readable digests.                                                                                                                                                                                           |
| **Lint**            | Health check: structural errors, missing evidence, broken links, contradictions, stale claims, open questions, duplicate claim IDs.                                                                                                                                                             |
| **Confidence**      | Trustworthiness of a claim (0.0–1.0). Initially estimated by the LLM, calibrated by compile via four-factor model. Page-level confidence = arithmetic mean of all claims.                                                                                                                       |
| **Identifier**      | Slug-based, stable, human-readable name for a page, claim, or other wiki entity. Page identifiers follow `{type}.{slug}`, claim identifiers `{page-id}#claim-{slug}`. See [identifier-spec.md](cross-cutting/identifier-spec.md).                                                              |
| **index.md**        | Content catalog — every page with link, one-liner summary, type, metadata, and claim ID registry. First point of entry for queries and update lookups.                                                                                                                                          |
| **log.md**          | Append-only chronicle of all operations. Parseable with Unix tools.                                                                                                                                                                                                                             |
| **Schema**          | AGENTS.md / wiki-schema.md — defines structure, conventions, and workflows for the LLM.                                                                                                                                                                                                         |
| **Dashboard**       | Auto-generated report page (`reports/`) visualizing the wiki's health status. Claims are referenced by `page-id#claim-id`.                                                                                                                                                                      |
| **Agent Digest**    | Machine-readable JSON file (`.exolith/cache/agent-digest.json`) with a compact summary of all wiki pages including claim IDs.                                                                                                                                                                  |
